# Ретрограден — контекст за Claude Code

Българско астрологично мобилно приложение. Прототип: единствен самостоятелен
файл `retrograden.html` (HTML+CSS+JS, без build стъпка, без бекенд).

## Философия на продукта
- USP: астрономически ГРУНТИРАН хумор и социална механика, НЕ предсказателна
  точност. Никога не позиционираме приложението като буквален инструмент за
  предсказване на бъдещето.
- Тон: ироничен, брутално честен, български градски контекст (НАП, Цариградско
  шосе, WhatsApp групи, наем, LinkedIn). Виж SIGNS/DAILY_POOL/HOUSE_MEANINGS
  масивите в кода за калибрирания глас.
- Говорител: "Ретрограден" — мистериозен глас, говори навсякъде в приложението,
  без изградена лична история (засега).

## Критично разграничение (не го чупи)
Астрономическата точност (позиции на планети, ретроградни периоди, фази на
Луната) е РЕАЛНА и верифицирана. Астрологичната ИНТЕРПРЕТАЦИЯ е за забавление
и самоанализ, не научно доказан метод. Всеки нов текст/функция трябва да пази
тази граница — виж дисклеймъра в наталния екран.

## Технически стек
- Единствена зависимост: `astronomy-engine` (JPL/VSOP ефемериди), зареждана
  през CDN (`cdn.jsdelivr.net/npm/astronomy-engine@2/astronomy.browser.min.js`),
  глобална `window.Astronomy`.
- Без framework, без bundler. Vanilla JS в един `<script>` таг.
- Персистентност: `localStorage` (ключ `retrograden_natal_v1`) — файлът се
  отваря директно през `file://`, затова НЕ ползвай `window.storage`
  (артефактно API, важи само в Claude.ai интерфейса, не и тук).

## Задължителна дисциплина при промени
1. **Никога не обявявай нещо за "готово" без изпълнена проверка.** Минимум:
   `node --check` за синтаксис + tag-balance проверка (section/article/div/
   button/script open==close броене). За логика — извличай точния код от
   файла (не преписвай наизуст) в Node harness и го изпълнявай с реални тестови
   стойности.
2. **Верифицирай астрономически/астрологични формули срещу независим източник**
   преди да ги вградиш — виж PROJECT_CONTEXT.md за вече намерените и сверени
   източници. Не компилирай географски координати или исторически факти по
   памет в големи партиди — рискът от грешка е реален (виж инцидента с 259-те
   града в PROJECT_CONTEXT.md).
3. **Една смислова промяна на итерация.** Не преструктурирай няколко системи
   едновременно.
4. При редакция на голям файл предпочитай `str_replace` с уникален многоредов
   контекст пред редакция по номер на ред (номерата се местят).
5. След всяка успешна редакция презареди файла, преди следваща редакция върху
   същата зона — кешираният преглед остарява.

## Известни граници на текущия прототип
